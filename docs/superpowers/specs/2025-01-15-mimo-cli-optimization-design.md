# MiMo CLI 全面优化设计文档

**日期**: 2025-01-15
**版本**: v2.0
**状态**: 待实现

## 1. 概述

### 1.1 目标
全面优化 MiMo CLI 的交互体验和对话功能，严格对齐 Claude Code 的交互模式，打造专业、流畅的终端 AI 编程助手体验。

### 1.2 范围
- 交互架构重构（中等重构）
- 对话功能全面增强
- 视觉风格严格对齐 Claude Code

## 2. 交互架构重构

### 2.1 当前架构问题
- `App.tsx` 承担了太多职责（状态管理、消息处理、工具调用、UI 渲染）
- 消息历史和对话状态耦合在一起，难以扩展
- 缺乏对话会话概念

### 2.2 新架构设计

```
src/
├── conversation/          # 新增：对话管理核心
│   ├── ConversationManager.ts  # 会话管理（创建、切换、持久化）
│   ├── MessageHistory.ts       # 消息历史（支持编辑、删除、回溯）
│   └── types.ts
├── ui/                    # 重命名：tui → ui
│   ├── components/        # UI 组件
│   │   ├── ChatArea.tsx       # 聊天区域（消息列表 + 流式输出）
│   │   ├── InputBar.tsx       # 底部输入栏（Claude 风格）
│   │   ├── StatusLine.tsx     # 顶部状态栏
│   │   ├── ToolPanel.tsx      # 工具调用展示
│   │   ├── DiffViewer.tsx     # 代码 diff 预览
│   │   ├── ThinkingIndicator.tsx  # 思考中动画
│   │   └── SessionSwitcher.tsx    # 会话切换
│   ├── hooks/
│   │   ├── useConversation.ts
│   │   ├── useStreaming.ts
│   │   └── useToolApproval.ts
│   └── App.tsx
```

### 2.3 核心模块职责

#### ConversationManager
- 管理多个对话会话
- 会话的创建、切换、删除、持久化
- 当前会话状态管理

#### MessageHistory
- 消息的增加、编辑、删除
- 消息回溯（回到某一步重新提问）
- 分支历史管理

## 3. 对话功能增强

### 3.1 多轮对话上下文管理
- 消息支持编辑、删除、重新发送
- 消息持久化到 `~/.mimo/sessions/`
- 会话列表展示，支持快速切换

### 3.2 思考过程可视化
- AI 调用工具时显示 "Viewing file..." / "Running command..." 状态
- 工具调用进度实时更新
- 支持取消当前操作（Ctrl+C）

### 3.3 代码编辑实时预览
- Write/Edit 工具结果以 diff 形式展示
- 语法高亮 + 行号显示
- 支持一键应用/撤销

### 3.4 对话分支/回溯
- 支持回到任意消息重新提问
- 分支历史可视化（类似 git 分支图）

### 3.5 会话管理
- 多会话并行
- 会话命名、归档、删除
- 自动保存/恢复

## 4. 视觉风格对齐 Claude Code

### 4.1 配色方案
- 背景：深色主题（接近 Claude Code 的 `#1a1a2e`）
- 用户消息：蓝色系（`#3b82f6`）
- AI 消息：绿色系（`#10b981`）
- 工具调用：黄色系（`#f59e0b`）
- 状态栏：灰色系（`#6b7280`）

### 4.2 布局

```
┌─────────────────────────────────────────┐
│  MiMo · Ready    ·  ~/project    [×]  │  <- 顶部状态栏
├─────────────────────────────────────────┤
│                                         │
│  You                                    │
│  帮我优化这段代码                        │
│                                         │
│  MiMo                                   │
│  好的，我来分析一下...                   │
│  [Viewing file: src/index.ts]           │
│  [Running: grep "function" ...]        │
│                                         │
│  这是优化后的代码：                       │
│  ```typescript                          │
│  // 优化后的代码...                      │
│  ```                                    │
│                                         │
├─────────────────────────────────────────┤
│  ❯ _                                   │  <- 底部输入栏
│                                         │
└─────────────────────────────────────────┘
```

### 4.3 动画效果
- 流式输出平滑滚动
- 工具调用状态切换动画
- 加载 spinner（Claude 风格）

## 5. 核心改进点

| 方面 | 当前 | 优化后 |
|------|------|--------|
| 消息管理 | 简单列表 | 支持编辑、删除、回溯 |
| 会话管理 | 无 | 多会话、持久化 |
| 工具展示 | 简单文本 | 实时状态 + diff 预览 |
| 输入体验 | 单行 | 多行、历史、快捷键 |
| 视觉风格 | 基础 | Claude Code 风格 |

## 6. 技术栈

- React 18 + Ink 4
- TypeScript 5
- OpenAI SDK
- chalk（颜色输出）
- cli-highlight（代码高亮）

## 7. 实现计划

1. **Phase 1**: 架构重构（ConversationManager, MessageHistory）
2. **Phase 2**: 核心对话功能（消息编辑、会话管理）
3. **Phase 3**: 工具展示优化（diff 预览、实时状态）
4. **Phase 4**: 视觉风格对齐（配色、布局、动画）
5. **Phase 5**: 测试与优化

## 8. 风险与考虑

- **兼容性**: 确保与现有 API 的兼容性
- **性能**: 大量消息时的渲染性能
- **用户体验**: 保持简洁，避免过度设计
