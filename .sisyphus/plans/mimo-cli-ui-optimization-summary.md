# MiMo CLI UI/UX 优化摘要

## 当前状态

MiMo CLI 的 UI 基于 Ink/React 构建，整体架构合理，但与 Claude Code 相比存在以下差距：

## 核心问题

### 1. 交互效率低
- 审批流程只有 y/n，无法批量授权
- 不支持外部编辑器输入长 prompt
- 无法滚动回看历史输出

### 2. 信息密度低
- 工具调用每条占 2-3 行，大量调用时滚动过快
- 同类工具调用不合并显示
- 长输出被截断后无法查看完整内容

### 3. 实时反馈不足
- Token 用量只在任务完成后显示
- 子代理执行过程无流式输出
- 缺少 chars/s 或 tokens/s 指标

### 4. 视觉一致性差
- 颜色方案使用默认值，缺乏品牌感
- 欢迎页面不适配不同终端尺寸
- 表格在窄终端会溢出

## 优化建议

### P0 - 核心体验 (必须做)

| 优化项 | 描述 | 工作量 |
|--------|------|--------|
| 审批增强 | 添加 always allow、批量审批 | 2h |
| 外部编辑器 | Ctrl+E 打开 $EDITOR | 3h |
| 滚动控制 | Ctrl+Up/Down 滚动历史 | 4h |
| Context 指示器 | 状态栏显示使用率 | 2h |
| 工具分组 | 同类工具合并显示 | 3h |

### P1 - 体验提升 (应该做)

| 优化项 | 描述 | 工作量 |
|--------|------|--------|
| 工具结果展开 | 点击查看完整输出 | 4h |
| 错误展示优化 | 结构化错误信息 | 3h |
| 实时指标 | tokens/s、成本实时显示 | 4h |
| 欢迎页面响应式 | 适配不同终端尺寸 | 3h |

### P2 - 视觉打磨 (可以做)

| 优化项 | 描述 | 工作量 |
|--------|------|--------|
| 颜色方案统一 | 建立品牌色彩系统 | 2h |
| Markdown 增强 | 代码块行号、表格响应式 | 4h |
| 历史搜索 | Ctrl+R 模糊搜索 | 4h |
| 自动补全 | Tab 补全文件路径 | 6h |

## 快捷键方案

### 全局
- `Ctrl+C` - 取消/退出 (已实现)
- `Ctrl+L` - 清屏
- `Ctrl+E` - 打开外部编辑器
- `Ctrl+R` - 搜索历史
- `Ctrl+Up/Down` - 滚动
- `PageUp/PageDown` - 翻页

### 输入框
- `Enter` - 发送 (已实现)
- `Shift+Enter` - 换行 (已实现)
- `Tab` - 自动补全
- `Ctrl+A/E` - 行首/行尾
- `Ctrl+K/U` - 删除到行尾/行首

### 审批
- `y/n` - 允许/拒绝 (已实现)
- `a` - 本次会话始终允许
- `Y` - 允许所有待审批

## 技术实现要点

1. **Ink 组件**: 使用 Box flex 布局，Text 样式属性
2. **状态管理**: useRef 保持引用，useCallback 优化回调
3. **性能**: 虚拟化长列表，React.memo 避免重渲染
4. **响应式**: useStdout() 获取终端尺寸，动态调整布局
